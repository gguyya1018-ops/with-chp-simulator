1. 전체 흐름 한눈에 보기

각 단계 설명

1단계: 데이터 입력

SMP(시간별), 판매실적, 외부거래, 용수약품, 공휴일, 전력수급 데이터와 단가/요금 테이블(CP, NG단가, 전기요금, 열요금, 수도/약품단가 등)을 Excel로 일괄 업로드하거나 개별 입력합니다. 필수 데이터: SMP, 외부거래, NG단가, CP/TLF.

2단계: 설정

CHP 성능표(부하별 열/전기/가스), PLB 성능표, 축열조(용량/초기잔량/최소유지), 기동조건(Hot/Warm/Cold), 정비일정, 외부수열원 레지스트리, 단가산정 계수, 전력수요 패턴(월별 평균kW + 24시간 패턴) 등 8개 스텝으로 나뉘어 설정합니다.

3단계: 데이터 산출

입력 데이터와 설정을 결합하여 중간 계산을 수행합니다. MP(정산가격) = SMP x TLF + 무부하비정산, 일별 용량요금(CP), 외부수열비용, 열판매수입, 용수약품비가 산출됩니다. 함수: runAllCalc → calculateMP, calculateCapacityCharge, calculateExternalHeatCost, calculateHeatSalesRevenue, calculateWaterChem.

4단계: 기동계획 (최적화)

365일 각각에 대해 CHP 가동시간/부하율을 결정하는 핵심 단계입니다. MPC(Model Predictive Control) 방식으로 매일 30일 앞을 DP로 탐색한 후 오늘의 결정만 확정하고, 내일 다시 계산합니다. 함수: solveMPC (동기), solveMPCAsync (비동기).

5단계: 비용산출

최적화 결과를 바탕으로 전력비(기본료/수전요금/기후환경/연료비/전력기금), 가스비, 용수약품비, 수열비를 월별/연간으로 집계합니다. 함수: renderAllCosts.

6단계: 운영손익

수입(전력판매 + 용량요금 + 열판매)에서 비용(연료비 + 기동비 + 수열비 등)을 차감하여 공헌이익을 산출합니다. 월별 차트와 일별 상세 보기를 제공합니다. 함수: renderContribution.

2. 최적화(MPC) 작동 원리

MPC란?

Model Predictive Control. 내비게이션에 비유하면: "목적지까지 30km 앞을 보고 지금 어느 길로 갈지 결정하되, 1km 가면 다시 30km 앞을 보고 다시 판단하는 것"입니다.

위드시뮬에서는 오늘부터 30일 앞까지를 DP(동적 계획법)로 분석하여 가장 비용이 적은 경로를 찾되, 오늘 결정만 확정합니다. 내일은 실제 축열조 잔량을 기준으로 다시 30일 앞을 탐색합니다. 이렇게 365일을 하루씩 진행하면 1년 전체 계획이 만들어집니다.

전체 365일을 한 번에 보는 DP(solveDP)도 있지만, MPC가 실시간 축열조 상태를 반영하므로 더 정확합니다. 현재 정밀 최적화는 MPC 방식을 사용합니다.

매일 반복하는 7단계

MPC 루프 안에서 하루마다 아래 7단계가 순서대로 실행됩니다. 함수 solveMPC 내부 구조입니다.

1
데이터 준비
365일 전체의 일별 데이터를 미리 구성합니다. 수요(시간별), 외부수열, SMP/MP, NG단가, 공휴일, 정비일 등을 하나의 배열로 통합합니다. 각 날의 CHP 부하별 비용(연료비 - 전기수익)도 사전 계산합니다.
입력: DATA 객체(전체 엑셀 데이터), 설정값
출력: dayInfo 배열 (365개, 날별 모든 정보 포함)
관련: buildDailyData
2
사전검증 (overflow 방지)
현재 축열조 잔량에서 CHP를 가동했을 때 축열조가 넘치지 않는 최대 안전 가동시간을 계산합니다. 24시간부터 역순으로 줄여가며, 어떤 시작시간이든 overflow가 없는 시간을 찾습니다. 이 값이 DP 탐색의 상한이 됩니다.
입력: 현재 축열조 잔량, 일별 수요 패턴, CHP 열출력
출력: 날/잔량별 최대 안전 시간 (0~24)
관련: calcMaxSafeHours
3
역방향 DP (30일 윈도우)
오늘부터 30일(horizon) 범위에서, 마지막 날부터 오늘로 거꾸로 계산합니다. 각 날/축열조 잔량/정지 상태 조합마다 "CHP를 몇 시간, 어느 부하로 돌려야 향후 총비용이 최소인가"를 구합니다. 상태 공간: 축열조(10Gcal 단위) x 정지레벨(4단계: ON/Hot/Warm/Cold).
입력: dayInfo, 사전검증 결과, DP 상태공간
출력: 오늘의 최적 결정 테이블 (잔량x정지레벨 → 시간+부하)
관련: DP 역방향 루프 (Bellman Equation)
4
결정 추출
현재 실제 축열조 잔량과 CHP 정지 상태를 기반으로, DP 결과 테이블에서 오늘의 최적 결정을 꺼냅니다. 결과: "CHP 18시간, 80% 부하" 또는 "CHP 0시간(정지)" 등.
입력: 실제 축열조 잔량, 정지레벨, DP 결정 테이블
출력: 오늘의 CHP 가동시간, 부하율
5
블록 배치 (Stage 2)
결정된 가동시간을 하루 24시간 중 어디에 배치할지 정합니다. MP(전력 판매가)가 높은 시간대에 CHP를 가동하면 수익이 극대화됩니다. 모든 시작시간을 시뮬레이션하여, overflow 없이 MP 합계가 최대인 배치를 선택합니다. 전날과의 연속성(최소정지시간)도 체크합니다.
입력: CHP 시간/부하, 시간별 MP, 축열조 잔량
출력: CHP 시작시각 (예: 6시~24시)
6
PLB 판단 (Stage 3)
CHP 배치 결과로 시간별 축열조를 시뮬레이션하여, 최소 잔량 아래로 떨어지는 시점이 있는지 확인합니다. 부족하면 보조보일러(PLB)를 부족 시점 직전에 투입합니다. 대수(최대 3대)와 가동시간은 부족분 기반으로 자동 계산됩니다.
입력: CHP 블록 배치, 시간별 수요/외부수열
출력: PLB 대수, 시간, 시작시각 (또는 미투입)
7
시간별 시뮬 (Stage 4)
0시부터 23시까지 한 시간씩 실제 운전을 시뮬레이션합니다. 매 시간: 축열조 = 이전잔량 + 외부수열 + CHP열 + PLB열 - 수요. 상한/하한 제약 적용, 시간별 전기 수익(MP x 발전MW) 계산. 이 날의 마지막 축열조 잔량이 내일 MPC 루프의 시작값이 됩니다.
입력: CHP 블록, PLB 블록, 시간별 수요/외부수열
출력: 24시간 상세 기록 (열/전력/축열조/비용), 일별 집계
이 7단계가 365일 동안 반복됩니다. 전체 소요시간은 약 3~10초이며, UI에서는 워크플로우 오버레이로 진행 상태를 시각화합니다.

3. 데이터 흐름도

시뮬레이터에 입력되는 데이터는 크게 입력 데이터(실적/시장)와 단가/요금 데이터로 나뉩니다. 모든 데이터는 전역 DATA 객체에 저장되며, 설정값은 DOM 요소에서 직접 읽습니다.

입력 데이터

SMP 시간별
계통한계가격. 365일 x 24시간. 전기 판매가의 기준.
사용: MP산출, DP비용계산, 블록배치
판매실적 (시간별)
용도별 열 판매량(주택/업무/공공). 시간별 수요 패턴의 원천.
사용: buildDailyData의 수요 계산
외부거래 (일별)
외부수열원별 일별 수열량 (남부소각장, ERG, SRF, 인테코, 안산 등).
사용: buildDailyData의 외부수열 계산, 수열비용
용수약품 (일별)
용수사용량(m3), 약품별 사용량(Kg). 비용산출의 입력.
사용: calculateWaterChem
공휴일
월/일/요일/공휴일여부. TLF 선택과 요일구분에 사용.
사용: MP산출, buildDailyData
전력수급 (시간별)
용량(capacity), 급전(dispatch), 열제약(heat_constraint). 시간별 데이터.
사용: 전력비 계산, 계획수전 산출

단가/요금 데이터

CP/TLF (월별)
평일CP, 휴일CP, 평일/토요일/공휴일 TLF(손실보전계수).
사용: MP산출(SMP x TLF), 용량요금 계산
NG단가 (월별)
NG단위열량, 부과금/부담금, CHP/PLB/FC 열량단가(원/MJ).
사용: CHP/PLB 연료비 계산 (DP 비용함수의 핵심)
전기요금표 (월별x시간별)
평일/토요일/공휴일별 시간대별 전기요금. 기본료, 기후환경, 연료비, 전력기금.
사용: 전력비(수전요금) 계산
열요금표 / 수도약품단가 (월별)
열 판매 단가(주택/업무/공공), 수도요금 구간별, 약품별 단가.
사용: 열판매수입, 용수약품비 계산

buildMaintenanceSet		 정비일엔 CHP 강제 정지 최대 3개 기간 설정 가능. 겹치면 자동 합산.
기동비용 조건 변경
(Hot/Warm/Cold)		 설정 Step 3 (hotTime, warmTime, coldTime, gasRate)
calcStartupCost 함수		 DP에서 ON/OFF 결정 변경 정지레벨: 0=ON, 1=Hot(1일), 2=Warm(2일), 3=Cold(3일+). 기동비 = 시간 x 가스량 x NG단가.
DP 탐색 정밀도 변경 solveMPC 내부 STEP 변수 (현재 10Gcal) 상태 수 변경 → 속도/정확도 트레이드오프 STEP 줄이면 정밀하지만 느려짐. STEP=5면 상태 2배, 시간도 약 2배.
CHP 성능표, 축열조 용량, DP 탐색 범위 등을 동시에 변경하면 영향이 복합적입니다. 한 번에 하나만 변경하고 결과를 검증하는 것을 권장합니다.

5. 30년 확장 로드맵

현재 상태

1년 (365일) 시뮬레이션. MPC horizon = 30일. DP 계산 약 3~10초.

데이터: 실적 기반 (엑셀 업로드). 모든 입력이 1년 단위.

30년으로 늘리면 무엇이 달라지는가

항목현재 (1년)30년비고
일수365일10,950일30배 증가
MPC horizon30일30일 (유지 가능)MPC 특성상 horizon은 고정해도 됨
MPC 루프 횟수365회10,950회단순 30배, 각 루프의 DP는 동일 크기
예상 소요시간3~10초90~300초 (1.5~5분)Web Worker 분리 시 UI 블로킹 없음
데이터 크기시간별: 8,760행시간별: 262,800행엑셀 단일 시트 한계 주의

핵심 과제: 30년치 데이터 확보

실적 데이터는 과거 1~2년뿐입니다. 30년 시뮬레이션을 위해서는 미래 데이터를 추정해야 합니다.

추정 방법
  • 수요: 기준년도 패턴 반복 + 연도별 성장률 적용 (설정의 demandGrowthPct 활용)
  • SMP: 기준년도 패턴 + 연도별 물가상승 반영, 또는 시나리오별 가정
  • NG단가: 기준년도 + 연도별 상승률 (에너지원 가격 시나리오)
  • 외부수열: 기준년도 패턴 반복 + 신규 수열원 반영 시점 설정
  • 기기 성능: 경년 열화 모델 적용 (효율 감소, 정비 주기 변경)

단계별 확장 방안

1단계: 데이터 생성기 구현
기준년도 실적을 복제하여 N년치 데이터를 자동 생성하는 함수 구현. 연도별 성장률/물가상승 파라미터를 설정 UI에 추가. buildDailyData의 연도 루프를 다년도로 확장.
2단계: 성능 최적화
MPC 루프를 Web Worker로 분리하여 UI 블로킹 방지. DP 내부 Float64Array 최적화는 이미 되어 있으므로, 루프 자체의 오버헤드만 관리. 진행률 표시를 10,950일 기준으로 조정.
3단계: 결과 집계/시각화 확장
월별 요약을 연별 요약으로 확장 (aggregateMonthly → aggregateYearly). 30년 추세 차트(연간 비용, 공헌이익 추이) 추가. NPV(순현재가치), IRR(내부수익률) 등 투자분석 지표 산출.
4단계: 시나리오 비교
복수 시나리오(낙관/기본/비관)를 설정하여 30년 결과를 비교하는 기능. 기기 교체 시점, 수열원 변경 등의 이벤트를 타임라인에 설정.
MPC의 장점: horizon 30일을 유지하면 DP 상태공간은 변하지 않습니다. 30년이라도 매일 30일짜리 소형 DP를 10,950번 풀 뿐이므로, 전체 365일 DP보다 오히려 확장에 유리합니다.

6. 용어 사전

MPC (Model Predictive Control)
모델예측제어. 매 시점 미래 N일을 예측하여 최적 결정을 내리되, 한 단계만 실행하고 다음 시점에 다시 예측하는 제어 방식.
DP (Dynamic Programming)
동적 계획법. 큰 문제를 작은 부분 문제로 나누어 최적해를 구하는 알고리즘. 여기서는 역방향으로 각 상태의 최소비용을 계산.
SMP (System Marginal Price)
계통한계가격. 전력시장에서 시간별로 결정되는 전기 도매가격. CHP의 전기 판매 수익 기준.
MP (Market Price / 정산가격)
실제 정산에 사용되는 가격. SMP x TLF(손실보전계수) + 무부하비정산으로 산출.
TLF (Transmission Loss Factor)
송전손실보전계수. 발전소에서 소비지까지 전력 손실을 보상하는 계수. 평일/토요일/공휴일별로 다름.
CP (Capacity Payment / 용량요금)
발전 용량 확보에 대한 고정 보상금. CHP가 가동하지 않아도 평일/휴일별로 받는 수입.
CHP (Combined Heat and Power)
열병합발전. 가스를 태워 열과 전기를 동시에 생산. 전기는 판매하고 열은 지역난방에 공급.
PLB (Peak Load Boiler)
첨두부하보일러. 열수요가 CHP만으로 부족할 때 보조적으로 가동. 전기는 생산하지 않음.
축열조 (Thermal Storage)
열을 온수 형태로 저장하는 탱크. 생산과 소비의 시간차를 완충. 최소잔량(storageMin)과 최대용량(storageCap)의 제약이 있음.
Horizon (탐색 범위)
MPC에서 미래를 내다보는 일수. 현재 30일. 이 범위 내에서 DP로 최적 경로를 탐색.
netRequired (순필요열량)
수요 - 외부수열(손실 차감) - 인테코 순수열. CHP/PLB/축열조가 충당해야 할 열량.
stopLevel (정지레벨)
CHP 정지 경과 상태. 0=가동중, 1=Hot(1일 정지), 2=Warm(2일), 3=Cold(3일+). 재기동 시 기동비용에 영향.
Bellman Equation
DP의 핵심 점화식. "현재 비용 + 미래 최적 비용 = 현재부터 끝까지 최적 비용". 역방향으로 채워감.
Overflow / Underflow
축열조가 상한(cap) 또는 하한(min)을 벗어나는 상황. overflow 시 열을 버려야 하고, underflow 시 열 공급 부족.
