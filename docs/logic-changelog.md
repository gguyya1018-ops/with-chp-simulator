## 1. 기준선

**날짜**: 2026-03-03
**상태**: 4단계 파이프라인 완성 직후

### 로직 요약
- Stage 1(DP): 일별 총량(`netRequired`) 기반 축열조 변화 계산
- Stage 1(휴리스틱): `roomForHeat = netReq + (storageCap - storageLevel)` 기반
- Stage 2: CHP 블록 배치 시에만 시간별 수요 사용 (MP 스코어링)
- PLB 배치: 항상 12시 중심 고정

### 주요 문제점
1. CHP 가동시간/부하 결정에 시간별 수요 패턴 미반영
2. 새벽 저수요/저녁 피크 같은 패턴을 무시
3. 축열조가 시간대별로 바닥을 칠 수 있는 상황 미감지
4. PLB가 항상 정오에 배치되어 실제 수요 피크와 불일치

## 2. v1: 시간별 수요 패턴 전면 반영

**날짜**: 2026-03-14

### 문제점 (기준선 대비)
- DP가 일별 총량으로만 축열조를 계산 → 시간대별 과부족 무시
- 휴리스틱 Pass 2가 `roomForHeat` 총량 기반 → 오버플로/언더플로 미감지
- PLB 12시 고정 → 수요 피크와 무관

### 수정 내용

#### 2-1. DP 내부 시간별 축열조 시뮬레이션 내장
**파일**: `js/app.js` solveDP()

**변경 전**:
```javascript
let newS = s + chpHeat - d.netRequired;  // 일별 총량
```

**변경 후**:
```javascript
// baseFlow[24] = hourlyExt + intecoH - demandH[t] 사전 계산
// CHP h시간: 모든 시작시간에 대해 24h 축열조 시뮬
// → MP 수익 최대 + overflow/underflow 최소 시작시간 선택
// → 시간별 축열조 최저점 기반 PLB 필요량 정밀 산출
```

#### 2-2. 휴리스틱 Pass 2: findMaxHours / checkStorageDeficit
**파일**: `js/app.js` runDailySimulation()

**변경 전**:
```javascript
const roomForHeat = Math.max(0, netReq + (storageCap - storageLevel));
const maxUsefulHours100 = Math.floor(roomForHeat / perf100.heatRate);
```

**변경 후**:
```javascript
// findMaxHours(perf, startH): 24h 시뮬로 오버플로 없는 최대 가동시간
// checkStorageDeficit(): CHP 없이 24h 축열조 최저점 계산
```

#### 2-3. PLB 배치: 수요 피크 대응
**변경 전**: `plbStartH = Math.round(12 - plbHours / 2)` (12시 중심 고정)
**변경 후**: 모든 시작시간 시뮬 → 축열조 최저점이 가장 높은 배치 선택

## 3. v2: 기동비용 경제성 반영

**날짜**: 2026-03-15

### 문제점 (v1 대비)
- DP가 ON/OFF 전환 시 기동비용을 고려하지 않음 → 빈번한 ON/OFF 허용
- 휴리스틱도 고정 기동비용(50,000원)으로 부정확한 판단
- 야간 단시간 정지(20시→0시 재기동) 같은 비합리적 패턴 발생
- 가스단가가 월별로 변동하는데 기동비용이 고정값

### 수정 내용

#### 3-1. 기동비용 산출 방식 변경
**변경 전**: 고정 기동비용 50,000원
**변경 후**: `기동시간(h) × 가스사용량(Nm³/h) × 월별 NG단가(원/Nm³)`

| 분류 | 조건 | 기동시간(h) | 가스사용량(Nm³/h) | 예시 비용 (500원/Nm³) |
|------|------|------------|-------------------|---------------------|
| Hot | 정지 ≤8h (1일) | 4 | 300 | 600,000원 |
| Warm | 정지 ≤48h (2일) | 6 | 350 | 1,050,000원 |
| Cold | 정지 >48h (3일+) | 8 | 400 | 1,600,000원 |

UI: 설정 > 기동조건 > 가스사용량(Nm³/h) 입력

#### 3-2. DP 상태 공간 확장
**변경 전**: `dp[축열조상태]` (1D, ~76 상태)
**변경 후**: `dp[축열조상태][정지레벨]` (2D, ~304 상태)
- 정지레벨: 0=가동중, 1=Hot, 2=Warm, 3=Cold
- 기동비용이 DP 비용함수에 포함 → 전역 최적에서 빈번한 ON/OFF 자동 억제

#### 3-3. 휴리스틱 연속운전 판단 개선
**변경 전**: `daysUntilRestart` (일 단위) vs `minStopTime` (시간 단위) — 단위 불일치 버그
**변경 후**: `hoursUntilRestart` (시간 단위)로 통일 + `calcStartupCost()` 적용
- 야간 단시간 정지 방지: prevChpEndH ≥ 16 && gap ≤ 8h → 연속운전 vs 재기동 비용 비교

### 시뮬 결과 비교
| 지표 | v1 | v2 | 변화 |
|------|-----|-----|------|
| DP 평균 생산단가 | - | - | - |
| 연간 총비용 | - | - | - |
| CHP 기동 횟수 | - | - | 감소 예상 |
| 야간 단시간 정지 횟수 | - | - | 감소 예상 |

> 시뮬 실행 후 채워넣을 것

### 특이사항
- DP 상태 수 4배 증가 (76→304)하나 계산시간은 여전히 수초 이내
- 첫날은 Cold 상태(stopLevel=3)로 시작 → 첫 기동 시 Cold 기동비용 반영

## 4. v3: 로직 버그 수정 및 UI/UX 개선

**날짜**: 2026-03-16

### 로직 수정

#### 4-1. CHP단가 계산식 오류 수정
**문제**: CHP단가가 음수(-130,000원/Gcal)로 표시 — 비현실적 값
**원인**: `chpElecRev`에 이미 전체 전기판매수익(MP×전력)이 포함되어 있는데, `cpCharge`(용량요금)까지 추가로 차감하여 이중공제 발생
**수정 전**: `(연료비 + 기동비 - 전기판매 - 용량요금) / 생산열`
**수정 후**: `(CHP연료비 + 기동비 - 열제약매출) / CHP생산열`
- 용량요금은 순비용에서만 반영 (단가 계산에서 제외)

#### 4-2. 휴리스틱 경로 기동비 누락 수정
**문제**: 휴리스틱 최적화 경로에서 `startupCost: 0` 하드코딩 → 기동비 미반영
**수정**: `heurStopDays` 추적 변수 추가, Hot/Warm/Cold 분류로 기동비 계산
```javascript
let heurStopDays = 99; // 초기: Cold 상태
// 매일 루프에서:
if (chpHours > 0 && heurStopDays > 0) {
    if (heurStopDays <= 1) { suTime = hotTime; suGas = hotGas; }
    else if (heurStopDays <= 2) { suTime = warmTime; suGas = warmGas; }
    else { suTime = coldTime; suGas = coldGas; }
    startupCost = suTime * suGas * ngCostNm3;
}
```

#### 4-3. 월별 집계 plbFuelCost 이중합산 수정
**문제**: `totals.plbFuelCost += b.plbFuelCost` 코드가 2번 실행
**수정**: 중복 제거

#### 4-4. 연료비 분리 표시
**변경 전**: 단일 "연료비" 컬럼
**변경 후**: CHP연료비 / 기동비 / 열제약매출 / CHP단가 / 용량요금 / PLB연료비 / PLB단가 / 순비용
- 월별·일별 테이블 모두 적용
- 계산 흐름(좌→우) 순서로 컬럼 배치

### UI/UX 개선

#### 4-5. 다크 테마 텍스트 선명도 개선
- `font-optical-sizing: auto`, `-webkit-font-smoothing: antialiased` 전역 적용
- 활성 탭 `font-weight` 600→500 (번짐 방지)
- muted 색상 밝기 상향: `--text-muted` #999→#aaa, `--text-faint` #888→#999
- 하드코딩 색상 일괄 상향 (#888→#999, #999→#aaa)

#### 4-6. 탭 디자인 계층 통일
- **Level 2 탭** (freq-tab, calcTabs, planTabs): step-btn 스타일 (배경 #141414, 보더 #333, 활성 시 accent 보더)
- **Level 3 탭** (inner-tab): pill/chip 스타일 (작은 둥근 모서리, 반투명 accent 배경)
- 설정 탭의 스타일을 기준으로 통일

#### 4-7. 제목 위치 통일
- `.section-bar` 고정 높이 40px → 모든 탭에서 제목 위치 일관
- `.section-title::before` 악센트 바 (3px × 15px) 통일
- 네비게이션 타이틀에 그래디언트 텍스트 + 📋 아이콘

#### 4-8. 기동계획 버튼 리디자인
- 분리형 rounded 버튼 (배경색 #141414, 보더 #333, gap 6px)
- 활성 시 accent 보더 + 밝은 텍스트 (설정 탭 step-btn과 동일 스타일)
- "로직 흐름" 버튼에 ❓ 도움말 아이콘(SVG) 추가

#### 4-9. 일별 세부 칩(배지) 리디자인
- 라이트 테마 배경색(#e8f5e9 등) → 반투명 다크 배경 (rgba 6%) + 미세 보더
- 텍스트 색상을 다크 팔레트에 맞춤 (녹 #6BCB77, 파 #7BA8F5, 주 #FFB74D, 보 #A78BFA)
- 호버 효과 추가

#### 4-10. 차트 범례 개선
- 둥글고 납작한 가로형 아이콘 (boxWidth: 16, boxHeight: 5, borderRadius: 3)
- 폰트 10px, 색상 #aaa, 패딩 14px

#### 4-11. 도넛 차트 축소
- 캔버스 높이 340→220px, aspectRatio: 1.8, cutout: 62%

#### 4-12. 설정 "값 적용" 기능
- 설정 변경 시 하단에 sticky 바 표시 (⚠ 설정이 변경되었습니다)
- "값 적용" 버튼 클릭 시 반영, "되돌리기" 버튼으로 원상복구
- 입력값 스냅샷 비교로 변경 감지
