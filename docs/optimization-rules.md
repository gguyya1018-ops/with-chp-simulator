# CHP/PLB 최적화 원칙

> 최적화 로직이 따라야 하는 불변 규칙.
> 코드 수정 시 이 원칙에 위배되는지 반드시 확인한다.
> 마지막 갱신: 2026-03-17

---

## 1. 절대 규칙 (위반 불가)

이 규칙들은 어떤 경우에도 깨지면 안 된다.

| # | 규칙 | 설명 |
|---|------|------|
| R1 | **축열조 하한** | 축열조 잔량은 어떤 시간에도 `storageMin` 아래로 내려가면 안 된다 |
| R2 | **축열조 상한** | 축열조 잔량은 어떤 시간에도 `storageCap`을 초과하면 안 된다. 열 버림(heat dump) 금지 |
| R3 | **열수급 균형** | 매 시간: 공급(CHP+PLB+외부수열) + 축열조방열 ≥ 수요. 열 부족 불가 |
| R4 | **최소운전시간** | CHP 가동 시 `chpMinRunHours` 이상 연속 운전 |
| R5 | **최소정지시간** | CHP 정지 시 `chpMinStopHours` 이상 연속 정지 |
| R6 | **정비일 CHP 불가** | 정비 기간 중 CHP 가동 금지 |
| R7 | **PLB 최소운전시간** | PLB 투입 시 `plbMinRunHours` 이상 가동 |

---

## 2. 최적화 목표 (우선순위 순)

우선순위가 높은 목표가 낮은 목표보다 먼저 충족되어야 한다.

| 우선순위 | 목표 | 설명 |
|---------|------|------|
| 1 | **안전** | 절대 규칙(R1~R7) 위반 없음 |
| 2 | **열공급 충족** | 365일 모든 시간의 열수요를 충족 |
| 3 | **비용 최소화** | 연간 총 운영비용(연료비+기동비-전기매출) 최소 |
| 4 | **PLB 최소화** | PLB 가동일수/시간 최소 (CHP 우선 원칙) |
| 5 | **축열조 안정** | 축열조 잔량을 적정 범위(30~80%)에서 유지 |

## 3. 판단 원칙

### 3-1. CHP 가동 판단
```
CHP를 돌릴지 말지는 다음 순서로 결정:
1. 정비일이면 → 가동 불가
2. 열이 필요한가? (netRequired > 축열조 여유)
   → 필요하면 → 가동 (비용과 무관하게 열공급 우선)
3. 열이 충분해도 경제적인가? (CHP 순비용 < 0, 즉 전기 팔아서 이익)
   → 이익이면 → 가동 (축열조 여유 범위 내에서)
4. 축열조가 가득 차면?
   → 부하를 낮추거나 시간을 줄여서 overflow 방지 (R2)
   → 그래도 안 되면 → 가동 중단
```

### 3-2. CHP 부하 판단
```
1. 기본: 경제성 최적 부하 선택 (costPerGcal 최소)
2. 축열조 여유가 부족하면 → 부하 낮춤 (열생산 감소)
3. 부하 하한: chpMinLoad (설정값)
4. 부하 낮춰도 overflow 불가피하면 → 가동시간 축소
```

### 3-3. CHP 시작시간 판단
```
1. MP(시장가격)가 높은 시간대에 배치 (전기 매출 극대화)
2. 단, overflow가 발생하는 배치는 제외 (R2)
3. 단, underflow가 발생하는 배치는 제외 (R1)
4. 전날 CHP 종료 → 오늘 CHP 시작 간격이 최소정지시간 이상 (R5)
```

### 3-4. PLB 투입 판단
```
1. CHP + 외부수열 + 축열조방열로 열수요를 충족할 수 있으면 → PLB 불필요
2. 충족 불가 시 → PLB 투입
   - 투입량: 부족분(deficit) 기반, 과투입 금지 (deficit의 2배 이하)
   - 투입 시간: 부족이 발생하는 시점 직전에 배치
   - 대수: 부족분을 최소 대수로 커버
3. PLB 최소운전시간으로 인해 과투입이 불가피한 경우
   → 1대로 낮추고, 부하도 낮춰서 과투입 최소화
```

### 3-5. 연속운전 판단
```
1. 오늘 CHP를 끄려는데, 며칠 후 다시 켜야 한다면?
   - 정지 기간 < 최소정지시간 → 끄지 못함, 최소부하로 유지 (R5)
   - 정지 기간 ≤ 3일 → 경제성 비교
     - 최소부하 유지 비용 vs 정지+재기동(기동비) 비용
     - 더 싼 쪽 선택
   - 정지 기간 > 3일 → 정지 허용
```

## 4. 검증 체크리스트

코드 수정 후 반드시 확인:

- [ ] R1: 모든 시간에 축열조 ≥ storageMin
- [ ] R2: 모든 시간에 축열조 ≤ storageCap (열 버림 0건)
- [ ] R3: 모든 시간에 열공급 ≥ 열수요
- [ ] R4~R5: 최소운전/정지시간 위반 0건
- [ ] R6: 정비일 CHP 가동 0건
- [ ] R7: PLB 최소운전시간 위반 0건
- [ ] PLB 과투입: PLB 열량 > deficit×2 인 날 0건
- [ ] 부하 다양성: 100% 외 부하가 선택되는 날 존재
- [ ] 총비용: 수정 전 대비 동등 또는 감소

## 5. 용어 정의

| 용어 | 정의 | 단위 |
|------|------|------|
| netRequired | 열수요 − 외부수열 − 인테코(손실 반영 후) | Gcal |
| roomForHeat | 열수요 + (축열조 상한 − 현재 잔량) | Gcal |
| deficit | storageMin − 축열조 최저 도달 잔량 | Gcal |
| costPerGcal | (연료비 − 전기매출) / 생산열량 | 원/Gcal |
| netCostH | 시간당 순비용 = 가스비/h − 전기매출/h | 원 |
| CHP단가 | (연료비 + 기동비 − 전력판매) / 생산열량 | 원/Gcal |
