# CHP 시뮬레이터 최적화 규칙

> 이 문서는 시뮬레이션 엔진의 판단 기준을 정의합니다.
> 조건을 수정하면 시뮬레이션 결과가 달라집니다.
> 지속적으로 업데이트하며 최적화를 개선합니다.

---

## 0. 최상위 원칙
- **목표: 연간 생산단가(원/Gcal) 최소화**
- 모든 판단은 비용 기준 (Merit Order)
- 1년치 SMP, 수요, 단가 데이터가 모두 주어져 있으므로 **예측이 아닌 최적 계획**을 수립

---

## 1. 열원별 비용 구조

### CHP
- 비용 = NG가스비 (원/h)
- 매출 = 전기판매(SMP × 발전량) + 열판매
- **CHP단가(원/Gcal) = (NG비용 - 전기수익) / 생산열**
- 단가가 음수 = 이익 (전기수익 > 가스비)

### PLB
- 비용 = NG가스비 (원/h)
- **PLB단가(원/Gcal) = NG비용 / 생산열**

### 외부수열
- 구매단가 고정

---

## 2. CHP 운전 규칙

### 2-1. 부하율 (Load)
- 정격: 100%
- **최소부하: 60%** → 이하로 내리면 효율 급락, 운전 불가
- 부하 범위: 60% ~ 100%
- 부하에 따라 열생산량, 발전량, 가스소비량 비례 변화

### 2-2. 연속운전 우선
- CHP는 **켰다껐다 반복을 최소화**
- 기동 1회마다 기동비용 발생 + 설비 스트레스
- 가능하면 **연속운전 + 부하 조절**로 대응
- 부하 조절이 불가능할 때만 정지

### 2-3. 최소운전시간 / 최소정지시간
- 최소운전시간: 설정값 (기본 8h) → 한번 켜면 최소 이 시간 운전
- 최소정지시간: 설정값 (기본 4h) → 한번 끄면 최소 이 시간 정지
- 정지시간 < 최소정지시간이면 → 연속운전으로 전환

### 2-4. 부하 변경 간격
- 부하를 급격히 올렸다 내렸다 하지 않음
- 부하 변경 시 최소 유지시간 적용 (향후 시간단위에서 구현)

---

## 3. 축열조 운영 규칙

### 3-1. 기본 제약
- 용량 상한: 설정값 (Gcal)
- 최소잔량: 설정값 (Gcal)
- 오버플로 절대 방지 → 축열조 가득 차면 CHP 시간/부하 줄임

### 3-2. 전략적 축열 관리
- **고SMP일 전에 축열조를 비워둠** → CHP 최대가동 여유 확보
- **저SMP일에는 축열 최소화** → PLB나 외부수열로 부족분 보충
- 축열조는 "SMP 차익거래" 도구:
  - 저SMP 시: 열 방출 (축열조 소진)
  - 고SMP 시: 열 저장 (CHP 최대가동 잉여열 축적)

### 3-3. 축열조 가득 찼을 때 대응 (우선순위)
1. CHP 부하를 60%까지 낮춤 → 연속운전 유지하며 열생산 줄임
2. 부하 60%에서도 넘치면 → CHP 정지 (최소정지시간 적용)

## 4. 글로벌 최적화 로직

### 4-1. 연간 스케줄링 순서
```
Step 1: 전체 365일의 SMP 평균 산출
Step 2: 각 일을 SMP 수준별 분류 (고/중/저)
Step 3: 고SMP일부터 CHP 최대가동 배정
Step 4: 축열조 잔량을 역추적하여 저SMP일의 CHP 시간 조정
Step 5: 연속운전 블록화 (기동/정지 최소화)
Step 6: 부하율 최적화 (축열조 제약 반영)
Step 7: PLB 잔여 수요 배분
```

### 4-2. SMP 기반 일별 우선순위
- 각 일의 SMP 평균을 기준으로 내림차순 정렬
- 고SMP일: CHP 최대가동 → 전기수익 극대화
- 저SMP일: CHP 축소 또는 정지 → 불필요한 가스비 절감
- 공휴일/주말: 수요 적고 SMP 낮으므로 CHP 축소 우선 검토

### 4-3. 연속운전 블록 최적화
- 단독 1일 기동보다 **연속 N일 기동**이 유리 (기동비용 절감)
- 연속 블록 내에서 부하율 변동:
  - 고SMP일: 100% 부하
  - 저SMP일(블록 내): 60~80% 부하 (정지보다 나은 경우)
- 블록 분리 판단: 정지 후 재기동 비용 vs 저부하 유지 비용 비교

### 4-4. 비용 비교 판단
모든 상황에서 **복수의 시나리오를 비교**하여 최저 단가 선택:
- 시나리오 A: CHP 연속운전 + 부하 조절
- 시나리오 B: CHP 정지 + PLB 가동
- 시나리오 C: CHP 부분부하 + PLB 병행
→ 각 시나리오의 총비용(원) ÷ 총생산열(Gcal) 비교 → 최저 선택

## 6. 솔버 비교

### 휴리스틱 (기존)
- 방식: 순방향 1-pass, SMP 기반 우선순위 + 룰 기반 판단
- 장점: 빠름, 직관적
- 한계: **미래를 보지 못함** → 주말에 불필요한 축열, 고SMP일 가동 부족

### DP 최적화 (신규)
- 방식: **Bellman Equation 기반 역방향 동적 프로그래밍**
- 상태: 축열조 잔량 (20Gcal 단위 이산화)
- 결정: 각 날의 CHP 시간 × 부하율
- **365일 전체를 역방향으로 계산** → 전역 최적해 보장
- 주말에 CHP를 안 돌리고 축열조를 비워두면 월요일에 더 벌 수 있다는 것을 자동 발견
- 계산시간: < 1초 (브라우저 내)
